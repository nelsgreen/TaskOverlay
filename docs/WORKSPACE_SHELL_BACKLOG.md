# Workspace shell backlog index

The accepted Workspace shell and task editor backlog is maintained in `WORKSPACE_SHELL_DIRECTION.md` until it is folded into the primary `BACKLOG.md`, `DECISIONS.md`, and `ROADMAP.md` after the active Design System sequence.

Current accepted direction:

- right-side Task Details remains the fast inspector on wide layouts;
- a focused Task Details modal is added inside Workspace;
- create/edit share one `TaskEditorContent` implementation;
- `Ctrl+Alt+Q` opens/focuses Workspace and starts task creation in the modal;
- Settings moves into a Workspace modal after parity;
- Workspace opens at startup after startup/backup checks;
- Overlay remains the attention layer;
- reminder delivery across native notifications and Overlay requires a separate reliability decision;
- Context actions become equal compact `Link`, `Hub`, and `Export` buttons.
